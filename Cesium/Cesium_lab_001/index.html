    2、本地开发环境¶

    2.1 使用Node创建服务¶

    Cesium 简单的开发方式是采用 Node.js，安装好Node.js后，去官网或者Github下载Cesium安装包：https://cesium.com/downloads/ 或 https://github.com/AnalyticalGraphicsInc/cesium/releases。目前最新为： Cesium-1.63.1.zip，下载后解压，可见如下目录：

    • App：官方自带的演示程序 HelloWorld；
    • Build：经过编译的Cesium服务，可直接部署到本地或远程服务器；
    • node_modules：Cesium的node.js依赖库；
    • Source：Cesium的源码；
    • Specs：Cesium的配置文件；
    • ThirdParty：第三方工具。

    使用编辑器（Windows下推荐VS Code）打开App下的 HelloWorld.html 文件，将内容替换成以下代码：

    <!DOCTYPE html>
<html lang="en">
<head>
  <!-- Use correct character set. -->
  <meta charset="utf-8">
  <!-- Tell IE to use the latest, best version. -->
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <!-- Make the application on mobile take up the full browser screen and disable user scaling. -->
  <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, minimum-scale=1, user-scalable=no">
  <title>Hello World!</title>
  <script src="../Build/Cesium/Cesium.js"></script>
  <style>
      @import url(../Build/Cesium/Widgets/widgets.css);
      html, body, #cesiumContainer {
          width: 100%; height: 100%; margin: 0; padding: 0; overflow: hidden;
      }
  </style>
</head>
<body>
  <div id="cesiumContainer"></div>
  <script>
    Cesium.Ion.defaultAccessToken = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJqdGkiOiJlZGVlM2E5Ni03ZmY3LTRhMjAtYTliNy1hNmE4ODI0NTI1NDAiLCJpZCI6MTgwNjIsInNjb3BlcyI6WyJhc2wiLCJhc3IiLCJhc3ciLCJnYyJdLCJpYXQiOjE1NzMyMDQ1NjV9.ZxHfmd9QRWpTg6tqolA20nwrSiBSYK4-ws4862T5itg';
    var viewer = new Cesium.Viewer('cesiumContainer');
    viewer._cesiumWidget._creditContainer.style.display="none";
  </script>
</body>
</html>

    其中：

    • Cesium.Ion.defaultAccessToken 为Cesium ion账户的Access Tokens，注册后即可获得。Access Tokens可采用默认值，也可自行另外添加，上述代码中的为自建值。（注：Cesium ion是一个3D内容的发布中心并且可以把你自己的数据进行切片、发布服务，Cesium和Cesium ion配合起来可以创建世界级3D地图程序。）
    • viewer._cesiumWidget._creditContainer.style.display="none"; 是为了取消视图中的Cesium官方Logo。

    保存修改，在解压后的Cesium安装包根目录下打开终端（推荐PowerShell），然后输入：node server，即可启动Cesium本地服务。

    PS C:\Users\user\Cesium161> node server
Cesium development server running locally.  Connect to http://localhost:8080/

    浏览器（推荐Chrome或Firefox）中打开：http://localhost:8080/，即可进入Cesium本地服务的主页面：

    这里有两个链接非常重要：Documentation 和 Sandcastle，Documentation 里面是Cesium的完整的API说明，里面可以找到：**1）**某一个模块的所有函数，属性；**2）**部分效果截图；**3）**部分函数，属性调用代码示例，缺点是搜索功能不够强大。Sandcastle 这个就厉害了，是一个沙盒，你可以在里面进行：**1）**浏览当前版本的一些功能特性；**2）**一个可运行的代码库；**3）**新建一个页面，进行代码测试；**4）**导出测试代码...等骚操作。推荐的学习方式是：

    1. 先浏览一遍沙盒里的所有示例，看看Cesium能做什么，能做成什么样，心里有数；
    2. 做自己需要的功能时，查找到相关示例代码，弄懂，最好浏览一遍用到的模块的API；
    3. 如果是深入研究的话，就需要有对WebGL有更深层次的了解；
    4. 熟悉Cesium运行机制后，在拿它做自己感兴趣的项目。

    2.2 使用http-server创建服务¶

    除了使用Node创建本地服务，还可使用http-server这个Node.js三方库创建本地服务。首先需要使用Node包管理命令nmp进行安装：

    npm install -g http-server # -g 表示全局安装

    新建文件夹 CesiumApp，再在该目录下新建 scripts 文件夹和 index.html 文件，并将 Build 下的 Cesium 复制到 scripts 文件夹内，随后在 index.html 中写入如下代码：

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta http-equiv="X-UA-Compatible" content="ie=edge">
    <title>Cesium App</title>
    <script src="./scripts/Cesium/Cesium.js"></script>
    <link rel="stylesheet" href="./scripts/Cesium/Widgets/Widgets.css">
    <style>
        html, body {
            height: 100%;
            margin: 0px;
            padding: 0px;
        }
    </style>
</head>
<body>
    <div id="cesiumContainer" style="height: 100%"></div>
    <script>
        // AccessToken访问Bing影像底图
        Cesium.Ion.defaultAccessToken = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJqdGkiOiJlZGVlM2E5Ni03ZmY3LTRhMjAtYTliNy1hNmE4ODI0NTI1NDAiLCJpZCI6MTgwNjIsInNjb3BlcyI6WyJhc2wiLCJhc3IiLCJhc3ciLCJnYyJdLCJpYXQiOjE1NzMyMDQ1NjV9.ZxHfmd9QRWpTg6tqolA20nwrSiBSYK4-ws4862T5itg';

        var viewer = new Cesium.Viewer('cesiumContainer');

        // 去除Cesium Logo
        viewer._cesiumWidget._creditContainer.style.display="none";
    </script>    

</body>
</html>

    CesiumApp 文件夹内打开终端，输入：

    http-server -p 8081 # http-server 可简写为 hs

    终端自动输出：

    Starting up http-server, serving ./
Available on:
  http://192.168.30.1:8081
  http://192.168.254.1:8081
  http://192.168.0.135:8081
  http://127.0.0.1:8081
Hit CTRL-C to stop the server

    然后在支持WebGIS的浏览器（推荐 Firefox 或 Chrome）中输入：http://127.0.0.1:8081/index.html（或者上面三个地址 + /index.html）即可进入Cesium的主界面。

    3、服务器开发环境¶

    3.1 创建Glitch项目¶

    所有Web程序需要一个Web服务器，Cesium也不例外。Cesium ion需要付费，学习阶段为了方便，可使用Glitch和 一个在线的IDE做程序开发。点击这里创建一个Glitch项目（推荐使用Github账号注册），Glitch加载之后，忽略模板提示，直接点击 index.html 删除整个内容，把以下代码粘贴进去：

    <!DOCTYPE html>
<html lang="en">
<head>
  <script src="https://cesiumjs.org/releases/1.63.1/Build/Cesium/Cesium.js"></script>
  <link href="https://cesiumjs.org/releases/1.63.1/Build/Cesium/Widgets/widgets.css" rel="stylesheet">
</head>
<body>
  <div id="cesiumContainer" style="width: 1520px; height:690px"></div>
  <script>
    // AccessToken访问Bing影像底图
    Cesium.Ion.defaultAccessToken = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJqdGkiOiJlZGVlM2E5Ni03ZmY3LTRhMjAtYTliNy1hNmE4ODI0NTI1NDAiLCJpZCI6MTgwNjIsInNjb3BlcyI6WyJhc2wiLCJhc3IiLCJhc3ciLCJnYyJdLCJpYXQiOjE1NzMyMDQ1NjV9.ZxHfmd9QRWpTg6tqolA20nwrSiBSYK4-ws4862T5itg';
    // 创建一个名为Viewer的Cesium控件
    var viewer = new Cesium.Viewer('cesiumContainer');
    // 去除Cesium Logo
    viewer._cesiumWidget._creditContainer.style.display="none";
  </script>
</body>
</html>

    Ctrl+S 保存后，点击左上角的预览图标，会打开一个新窗口，并显示了Cesium的主界面。随后若修改 index.html，这个弹出的窗口会自动刷新。

    • Spacefan：https://seed-gigantoraptor.glitch.me/
    • SciWorm：https://burly-tungsten.glitch.me/

    3.2 代码分析¶

    首先在HTML的head标签内引用CesiumJS的库，其中 releases 后面的版本号可更新至最新的Cesium版本。

    <script src="https://cesiumjs.org/releases/1.63.1/Build/Cesium/Cesium.js"></script>
<link href="https://cesiumjs.org/releases/1.63.1/Build/Cesium/Widgets/widgets.css" rel="stylesheet">

    创建一个HTML标签去承载CesiumJS控件（widget）：

    <div id="cesiumContainer"></div>

    从你的Cesium ion账户提供一个token去访问Bing影像底图，注意将 your_access_token 替换。

    Cesium.Ion.defaultAccessToken = 'your_access_token';

    最后，创建一个名为Viewer的Cesium控件，并且让他去使用上面定义的HTML元素：

    var viewer = new Cesium.Viewer('cesiumContainer');

    3.3 添加Cesium全球地形¶

    Cesium全球地形是一个高精度地形库，在你的Cesium ion账户里已经有访问权限。用下述代码替换我们做的第一个示例里的创建Viewer的部分:

    var viewer = new Cesium.Viewer('cesiumContainer', {
    terrainProvider: Cesium.createWorldTerrain()
});

    接下来，既可在预览界面缩放到特定位置，比如搜索 Grand Canyon, AZ, USA，就看到如下图效果（鼠标中键 或 Ctrl+鼠标左键 调整视角）：

    4、加载影像和地形数据¶

    打开 Documentation，搜索 viewer 类（本地网址：http://localhost:8080/Build/Documentation/Viewer.html?classFilter=viewer）可见该类有很多参数，且设置有默认值。 对于加载自定义影像数据而言， viewer 类的两个参数较为重要：

    • 一个是 baseLayerPicker 参数：它控制影像的拾取，在加载自定义影像前需将其值更改为 fasle，更改后右上角的影像切换按钮就会消失。
    // 创建一个名为Viewer的Cesium控件, 并加载Cesium全球地形数据
var viewer = new Cesium.Viewer('cesiumContainer', {
    baseLayerPicker: false,
    terrainProvider: Cesium.createWorldTerrain()
});

    • 第二个是 imageryProvider 参数：默认情况下为 createWorldImagery()，点击对应 Type 下的 ProviderViewModel可见有多种影像类型：

    • Demo:

      • Cesium Sandcastle Imagery Layers Demo
      • Cesium Sandcastle Imagery Manipulation Demo

    • See:

      • ArcGisMapServerImageryProvider
      • BingMapsImageryProvider
      • createOpenStreetMapImageryProvider
      • createTileMapServiceImageryProvider
      • GoogleEarthEnterpriseImageryProvider
      • GoogleEarthEnterpriseMapsProvider
      • GridImageryProvider
      • MapboxImageryProvider
      • MapboxStyleImageryProvider
      • SingleTileImageryProvider
      • TileCoordinatesImageryProvider
      • UrlTemplateImageryProvider
      • WebMapServiceImageryProvider
      • WebMapTileServiceImageryProvider

    其中，GoogleEarthEnterpriseImageryProvider 是加载 GoogleEarth 企业版影像数据，UrlTemplateImageryProvider 是根据 URL 模板加载影像数据，打开 GoogleEarthEnterpriseImageryProvider 就可以发现它是一个构造函数。由于 GoogleEarth 企业版影像数据已经无法访问，故采用 URL 模板加载谷歌中国地区的影像数据。将下面的代码替换 JavaScript 中的代码，即可实现谷歌中国地区的影像数据的加载：

    <script>

    // 创建谷歌中国地区的影像数据
    var guge = new Cesium.UrlTemplateImageryProvider({
      url:'http://www.google.cn/maps/vt?lyrs=s@800&x={x}&y={y}&z={z}',  
      tilingScheme:new Cesium.WebMercatorTilingScheme(),              
      minimumLevel:1,            
      maximumLevel:20
    });

  // 设置参数, 加载谷歌中国地区的影像数据
    var viewer = new Cesium.Viewer('cesiumContainer', {
      baseLayerPicker: false,
      imageryProvider: guge,
    });

    // 去除Cesium Logo
    viewer._cesiumWidget._creditContainer.style.display="none";

  </script>

    二、Cesium Ion入门¶

    1、Cesium云服务简介¶

    1.1 前言¶

    所有行业都知道云是未来或者现在的趋势，但是真正的完完全全提供地理信息云服务的恐怕只有 Google 一家，然而今天我居然发现 Cesium 提供了云服务，你没有看错，就是曾经的开源 3D 渲染框架。本文简单介绍之。

    1.2 Cesium Ion¶

    Cesium 是一款优秀的开源 3D 地图渲染引擎，这点是毋庸置疑的，然而前几天 Cesium 给我推送了一封邮件，一直忙着无暇顾及，今天打开来一看：呵，这家伙居然从一个框架变成了云服务提供商！

    其实现方式就是通过这个 Ion，在保持其原有渲染引擎不变的情况下，为每位用户提供了 5G 的存储空间，用户可以上传自己的数据（影像或者是 3D 数据）， Cesium 将此数据切割后存放在空间中，用户便可通过极其简单的方式分享和调用此数据。 Cesium 还为每位用户提供了几种默认影像数据和 1 种地表数据。详情见（https://cesium.com/ion/info/welcome）。

    前台调用方式基本不变（先将 Cesium 升级至最新版），调用云端数据也十分方便，如下：

    viewer.imageryLayers.addImageryProvider(
    new Cesium.IonImageryProvider({ assetId: 2 })
);

    即只需要改一下 assetId 号即可，当然需要先创建 Cesium 账户，并取到 Access Token，在 js 开篇加入：

    Cesium.Ion.defaultAccessToken = 'your/ion/access/token';

    1.3 启示¶

    这件事情看上去好像并没有多么神奇，但是我个人觉得这为我们建设**空间数据云**提供了一些启示。

    之前的数据中心（空间）建设方案中同样是以数据为主，但是我们并没有梳理出数据与服务的关系，大部分情况下要么是以数据本身为服务，提供查询、共享、下载等功能，要么是以瓦片为服务对外提供瓦片 api，使得别人能够通过这种规则调用瓦片数据。以上两种方式均没有将数据抽象成方便的服务，从而数据云的概念只是一个数据存储仓库，相对传统方式并没有本质的改变，只是增加了存储能力和安全性等等。而 Cesium ion 将瓦片数据做了封装，我们无需再关心底层的瓦片是如何构成的，作为用户也不需要知道数据是怎么被处理的，只需要上传数据，系统自动处理完后返回一个编号，用户用此编号就能在各处使用此数据，这是真正的数据云。

    1.4 总结¶

    本文简单介绍了 Cesium Ion，并简单分析了其对建设空间数据云的一点启示。
